## 🎯 실습 목표

1. **전문서비스 현황 분석** - 현재 서비스 제공 프로세스와 AI 도입 현황 진단
2. **경쟁사 벤치마킹** - 동종 업계 AI 활용 사례 분석 및 벤치마킹
3. **자사 역량 데이터 기반 진단** - 실제 기술 스택, 프로젝트 데이터로 AI 도입 기회 식별
4. **AI 기반 서비스 혁신** - 제안서 자동화와 코드 리뷰 시스템 구축

## 🚀 실습 워크플로우

### Phase 1: 서비스 현황 분석 (25분)
```bash
# 1. 현황 분석 팀 구성
/team 4:service_analysis --roles="business_analyst,service_manager,technical_lead"

# 2. 서비스 포트폴리오 분석
/autopilot "현재 제공 중인 서비스 포트폴리오를 분석. 서비스 범위, 품질 기준, 고객 만족도, 수익성 분석."

# 3. 기술 스택 및 프로세스 분석
/ralph "팀의 기술 스택과 개발 프로세스를 분석. 기술 역량, 개발 방법론, 품질 관리, 협업 툴 분석."
```

### Phase 2: 경쟁사 벤치마킹 (30분)
```bash
# 1. 벤치마킹 팀 구성
/team 4:benchmark_analysis --roles="market_researcher,competitor_analyst,trend_specialist"

# 2. 경쟁사 서비스 모델 분석
/autopilot "주요 경쟁사들의 서비스 모델, 가격 정책, 기술 스택을 분석. 성공 사례와 실패 사례 비교."

# 3. 기술 트렌드 및 베스트 프랙티스 도출
/ccg "전문서비스 분야의 최신 기술 트렌드를 분석하고 베스트 프랙티스를 도출. 클라우드 네이티브, 마이크로서비스, AI/ML 서비스, 데브옵스 분석."
```

### Phase 3: 자사 역량 기반 진단 (35분)
```bash
# 1. 역량 진단 팀 구성
/team 4:capability_assessment --roles="technical_assessor,project_manager,skill_analyst"

# 2. 기술 역량 매핑
/autopilot "팀의 기술 역량을 체계적으로 분석하고 매핑. 프론트엔드, 백엔드, DevOps, 데이터, 보안, AI/ML 역량을 포함한 다각적 분석."

# 3. 서비스 제공 역량 평가
/ralph "현재 역량과 시장 요구사항의 갭(Gap)을 분석. 역량 강화 방안, 신규 서비스 개발 가능성, 파트너십 전략을 제안."
```

### Phase 4: AI 기반 서비스 혁신 (30분)
```bash
# 1. 혁신 팀 구성
/team 4:innovation_team --roles="ai_specialist,service_designer,automation_expert"

# 2. AI 기반 제안서 자동화
/ccg "고객 요구사항을 기반으로 제안서를 자동 생성하는 AI 시스템을 설계. 요구사항 분석, 기술 스펙 추론, 제안서 자동 생성, 품질 관리 포함."

# 3. 코드 리뷰 자동화 시스템
/autopilot "개발된 코드의 품질을 자동으로 검토하고 피드백을 생성하는 시스템을 설계. 코딩 표준, 보안 취약점, 성능 최적화, 문서화 수준 자동 검출."

# 4. 서비스 혁신 로드맵 수립
/plan "분석된 역량과 시장 동향을 기반으로 AI 기반 서비스 혁신 로드맵을 수립. 단기 목표, 기술 스택 강화, 신규 서비스 개발, 시장 확대 전략 포함."
```

## 💡 전문서비스/IT 특화 프롬프트 10개

### 1. 서비스 포트폴리오 분석 프롬프트
```
[역할] 너는 전문서비스 포트폴리오 분석 전문가이다.
[맥락] 현재 제공 중인 서비스 포트폴리오를 분석하고 있다.
[명령] 서비스 범위, 품질 기준, 가격 정책, 서비스 레벨(SLA)을 분석하고 현재 포트폴리오의 강점과 약점을 식별하라. 시장 경쟁력, 고객 만족도, 수익성을 고려한 포트폴리오 개선 방안을 제안하라.
[형식] 포트폴리오 분석 보고서를 작성하고, 현재 현황, 강점/약점 분석, 개선 방안, 경쟁 우위 확보 방안을 포함하라.
```

### 2. 기술 스택 분석 프롬프트
```
[역할] 너는 기술 스택 분석 전문가이다.
[맥락] 팀의 기술 스택과 역량을 분석하고 있다.
[명령] 프론트엔드, 백엔드, DevOps, 데이터, 보안, AI/ML 기술 스택의 역량을 다각적으로 분석하고 매핑하라. 기술 수준, 경험 연수, 인증 여부, 프로젝트 경험을 고려한 종합적인 기술 스택 분석을 수행하라.
[형식] 기술 스택 분석 보고서를 작성하고, 각 기술 분야별 역량 현황, 강점, 약점, 개선 방안, 신규 기술 도입 계획을 포함하라.
```

### 3. 서비스 프로세스 분석 프롬프트
```
[역할] 너는 서비스 프로세스 분석 전문가이다.
[맥락] 서비스 제공 프로세스를 분석하고 있다.
[명령] 영업 프로세스, 개발 프로세스, 운영 프로세스, 고객 지원 프로세스를 분석하고 각 프로세스의 효율성과 문제점을 식별하라. 프로세스 최적화, 자동화 가능성, 협업 방안을 제안하라.
[형식] 프로세스 분석 보고서를 작성하고, 각 프로세스의 현황, 문제점 분석, 개선 방안, 자동화 전략을 포함하라.
```

[형식] 고객 만족도 분석 보고서를 작성하고, 만족도 현황, 영향 요인 분석, 개선 방안, KPI 설정을 포함하라.

[형식] 수익성 분석 보고서를 작성하고, 수익성 현황, 문제점 분석, 개선 방안, 최적화 전략을 포함하라.
